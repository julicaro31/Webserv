Configuration Manager (Parsing First!)

  • Parses the NGINX-style configuration file.
  • Extracts important data:
    • Server blocks (listening ports, hostnames).
    • Routes (static files, CGI paths).
    • Error pages.
    • Client limits (timeouts, max body size).
  • Stores settings in structured objects.
  • Passes configuration data to the Server Manager.

Server Manager

  • It uses the parsed configuration to initialize the server.
    • Using Server.setupSocket() to create a server, and store them in Vector.
  • Save all server FDs for POLLING.
  • Uses poll() to monitor:
    • New connections (POLLIN event on listening socket).
      • accept a new client, set it for non-blocking.
      • Set the events to POLLIN for reading only, we will set POLLOUT when we have a response to send.
    • Incoming data from connected clients (POLLIN).
    • Ready-to-write sockets (POLLOUT).
      • sending responses as chunks
  • HandleClientRequest:
    • Read received requests as chunks, and when it’s done, sent it to HTTP Request Handler
  • handleSigint: to end webserver by sigint properly

Server (core):

  • Includes core function for the server
    • Getters, and setters to create a server.
    • socket(), bind(), and listen() to sockets.

HTTP Request Handler

  • Parses incoming HTTP requests.
  • Extracts:
    • Headers, method (GET/POST/DELETE).
    • Target route (path, parameters).
    • Request body (for POST, PUT).
  • Passes the parsed request to the Router.
  • Router & Response Generator
    • Matches the request to a server block and route (based on config).
    • Serves static files (HTML files).
    • Forward to CGI Handler if it's a dynamic request.

CGI Handler

  • Uses fork() and execve() to execute external scripts (Python, PHP).
  • Communicates with the server using pipes.
